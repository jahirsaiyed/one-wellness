# API Specification — GCC Wellness Platform

**Base URL (staging):** `https://api-staging.gcc-wellness.com`
**Base URL (production):** `https://api.gcc-wellness.com`
**Protocol:** HTTPS only. HTTP requests redirected to HTTPS.
**Format:** JSON request/response bodies. Content-Type: `application/json`.
**Auth:** Bearer JWT in `Authorization` header OR httpOnly cookie `access_token`.
**Versioning:** `/v1/` prefix on all routes. Breaking changes introduce `/v2/`.

---

## Authentication & Authorization

### JWT Roles
| Role | Value |
|---|---|
| Client | `client` |
| Therapist | `therapist` |
| HR Admin | `hr_admin` |
| Platform Admin | `platform_admin` |

All protected endpoints return:
- `401 Unauthorized` — missing or expired token
- `403 Forbidden` — valid token but insufficient role
- `422 Unprocessable Entity` — request body validation failed (Pydantic)

## 1. Auth Service (`/v1/auth`)

### POST `/v1/auth/register`
Register a new user account.

**Auth:** None

**Request:**
```json
{
  "email": "user@example.com",
  "password": "SecurePass123!",
  "full_name": "Layla Ahmed",
  "preferred_language": "ar",
  "timezone": "Asia/Dubai",
  "intake_id": "uuid-optional"
}
```

**Response 201:**
```json
{
  "user_id": "uuid",
  "email": "user@example.com",
  "role": "client",
  "access_token": "jwt.access.token",
  "refresh_token": "jwt.refresh.token",
  "token_type": "bearer",
  "expires_in": 900
}
```

**Errors:** `400` email already registered | `422` validation error

---

### POST `/v1/auth/login`
Authenticate and receive tokens.

**Auth:** None

**Request:**
```json
{
  "email": "user@example.com",
  "password": "SecurePass123!"
}
```

**Response 200:**
```json
{
  "access_token": "jwt.access.token",
  "refresh_token": "jwt.refresh.token",
  "token_type": "bearer",
  "expires_in": 900,
  "requires_totp": true
}
```

> If `requires_totp: true`, client must call `POST /auth/totp/verify` before tokens are valid.

---

### POST `/v1/auth/token/refresh`
Rotate refresh token and issue new access token.

**Auth:** Refresh token in httpOnly cookie OR `refresh_token` body field.

**Response 200:**
```json
{
  "access_token": "new.jwt.token",
  "refresh_token": "new.refresh.token",
  "expires_in": 900
}
```

**Errors:** `401` invalid or expired refresh token (old token invalidated; re-login required)

---

### GET `/v1/auth/google/authorize`
Redirect to Google OAuth2 consent screen.

**Auth:** None

**Query Params:**
- `state` (required): CSRF protection token generated by client

**Response:** 302 redirect to Google

---

### GET `/v1/auth/google/callback`
Handle Google OAuth2 callback. Upserts user, returns JWT.

**Auth:** None

**Query Params:** `code`, `state`

**Response 200:**
```json
{
  "access_token": "jwt.access.token",
  "refresh_token": "jwt.refresh.token",
  "is_new_user": true
}
```

---

### POST `/v1/auth/totp/setup`
Generate TOTP secret and QR code URI. Requires authenticated session without TOTP confirmed.

**Auth:** Bearer (any role, pre-TOTP)

**Response 200:**
```json
{
  "secret": "BASE32ENCODEDSTRING",
  "qr_uri": "otpauth://totp/GCCWellness:user@example.com?secret=...&issuer=GCCWellness",
  "backup_codes": ["AAAA-BBBB", "CCCC-DDDD", "..."]
}
```

---

### POST `/v1/auth/totp/verify`
Verify TOTP code and mark 2FA confirmed in session.

**Auth:** Bearer (pre-TOTP)

**Request:**
```json
{ "code": "123456" }
```

**Response 200:**
```json
{ "verified": true, "access_token": "upgraded.jwt.token" }
```

**Errors:** `400` invalid code | `429` too many attempts

---

### POST `/v1/auth/logout`
Revoke refresh token.

**Auth:** Bearer

**Response 204:** No content

---

### POST `/v1/users/me/delete-request`
Initiate account deletion. Soft-deletes immediately; async purge within 30 days.

**Auth:** Bearer (client)

**Response 202:**
```json
{
  "message": "Deletion scheduled. All health data will be purged within 30 days.",
  "scheduled_deletion_at": "2026-06-01T00:00:00Z"
}
```

## 3. AI Companion (`/v1/companion`)

### POST `/v1/companion/chat`
Send message to AI companion. Returns Server-Sent Events (SSE) stream.

**Auth:** Bearer (or anonymous, limited to 3 messages)

**Request:**
```json
{
  "thread_id": "uuid-or-null-for-new-thread",
  "message": "أشعر بالقلق الشديد من العمل",
  "language": "ar"
}
```

**Response 200 (SSE stream):**
```
data: {"type": "token", "content": "أفهم"}
data: {"type": "token", "content": " تماماً"}
data: {"type": "crisis_check", "risk_level": "none"}
data: {"type": "done", "thread_id": "uuid", "message_id": "uuid"}
```

> If `risk_level` is `high`, the SSE stream is terminated and replaced with:
```
data: {"type": "crisis_override", "risk_level": "high", "emergency_numbers": {"UAE": "800-4673", "police": "999"}, "helpline_message": "..."}
```

**Errors:** `403` anonymous limit reached (return registration gate payload)

---

## Error Response Format

All errors follow this shape:

```json
{
  "error": {
    "code": "SLOT_ALREADY_BOOKED",
    "message": "The selected time slot is no longer available.",
    "details": null
  },
  "request_id": "uuid"
}
```

### Standard Error Codes

| HTTP | Code | Description |
|---|---|---|
| 400 | `VALIDATION_ERROR` | Request body fails Pydantic validation |
| 400 | `INVALID_HMAC` | Webhook signature mismatch |
| 401 | `TOKEN_EXPIRED` | JWT access token has expired |
| 401 | `INVALID_TOKEN` | Token cannot be decoded |
| 403 | `INSUFFICIENT_ROLE` | Role lacks permission |
| 403 | `CONSENT_REQUIRED` | Resource requires user consent |
| 404 | `NOT_FOUND` | Resource does not exist |
| 409 | `SLOT_ALREADY_BOOKED` | Concurrent booking conflict |
| 409 | `DUPLICATE_ENTRY` | Unique constraint violation |
| 429 | `RATE_LIMIT_EXCEEDED` | Too many requests |
| 502 | `AI_PROVIDER_ERROR` | Upstream AI provider unavailable |
| 503 | `SERVICE_UNAVAILABLE` | Maintenance window |
